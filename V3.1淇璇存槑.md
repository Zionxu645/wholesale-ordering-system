# V3.1 修复说明

本次只修复时间与生产单显示，不改变现有业务流程：

- 新询价单号和新订单号按中国标准时间（Asia/Shanghai）生成日期。
- 仪表盘“今日询价”和“本月订单”按中国时间统计。
- 新商品自动款号中的日期按中国时间生成。
- 生产单状态显示中文，例如 `已送达`，不再显示 `delivered`。
- 生产单打印时间固定按中国时间显示。
- 生产单 JSON 同时保留英文状态代码，并新增 `status_label` 与 `print_time_local`。

## 升级顺序

1. 在 Supabase SQL Editor 运行 `migration-v3.1-timezone.sql`。
2. 将本项目代码替换到 GitHub 仓库并推送。
3. 等待 Render 显示 `Live`。
4. 新建一条测试询价并转单，确认编号日期与中国当天一致。

旧测试单号不会自动改名；修复只影响升级后的新询价单和新订单。
