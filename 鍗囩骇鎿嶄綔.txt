当前线上系统已经有旧版数据库，因此按这个顺序升级：

第一步：先不要上传 GitHub。
在 Supabase → SQL Editor 中运行 migration-v3.sql 全部内容。
看到 Success. No rows returned 后再继续。

第二步：将本文件夹里的全部文件复制到 GitHub 仓库文件夹，保留原来的 .git 文件夹。
在 GitHub Desktop 提交并 Push。

第三步：等待 Render 自动部署为 Live。
本次不需要修改现有 SUPABASE_URL、SUPABASE_SERVICE_ROLE_KEY、JWT_SECRET、ADMIN_PHONE、ADMIN_PASSWORD。

第四步：进入 /admin 测试新增商品和上传图片。

注意：
- 不要在现有数据库重新运行 schema.sql。
- 不要上传 .env 或 node_modules。
- migration-v3.sql 不会删除旧客户、旧订单、商品和库存。
