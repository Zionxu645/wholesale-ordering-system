# 本次修复说明

## 已删除的旧架构

原项目同时存在 JSON 文件数据库、SQLite 风格 SQL 和 Supabase 后端，接口彼此不匹配。本版本不再使用：

- `db.js`
- `data/wholesale.json`
- 旧版 SQLite/MySQL 风格 `schema.sql`
- 被 Git 跟踪的 `node_modules`
- 错误创建成目录的 `.gitignore`

## 已完成

1. 前后端接口统一。
2. Supabase PostgreSQL 表结构统一。
3. 手机号注册和 JWT 登录。
4. 客户购物车、订单和数据隔离。
5. 管理员权限校验。
6. 商品、SKU、库存和价格管理。
7. PostgreSQL 事务下单，防止并发超卖。
8. 订单状态流转和取消归还库存。
9. ERP 仪表盘、客户管理、订单管理。
10. 生产单 JSON 与临时令牌打印页。
11. Render 环境变量和健康检查。
12. 所有异步接口统一错误捕获。
13. `.gitignore`、`package-lock.json` 和 Node.js 版本固定。
14. 本地语法检查、启动测试和依赖安全审计。

## 已验证

- `npm run check`：通过
- `npm test`：通过
- `npm audit --omit=dev`：0 个已知漏洞
- 未配置数据库时：服务保持启动，不会直接 crash

## 尚需用户提供的外部配置

代码本身已经修复，但真正连接数据库前必须：

1. 在 Supabase SQL Editor 执行 `schema.sql`。
2. 在 Render 配置 Supabase URL、service_role key、JWT secret 和管理员账号。
3. 部署后按实际业务录入商品、SKU、库存和价格。
